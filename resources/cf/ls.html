ls

Filter

Show hidden files
Do not ignore entries starting with '.', except '.' and '..'
List directory entries instead of directory contents. Do not dereference symbolic links
Recurse into subdirectories

Format

With -l, print the author of each file
Units to use
For each file, write the file's file serial number
Indicate the total number of file system blocks consumed by each file displayed
Assume block size of 1024 bytes
Evaluate the file information and file type for symbolic links specified on the command line to be those of the file referenced by the link, and not the link itself; however, ls shall write the name of the link itself and not the file referenced by the link
Multicolumn output (short entries)
Comma-separated list of short entries

Sort

Sort with the primary key being file size (in decreasing order) and the secondary key being filename in the collating sequence (in increasing order)
Sort with the primary key being time modified (most recently modified first) and the secondary key being filename in the collating sequence
List the entries in directory operands in the order they appear in the directory. Filter option -a will be turned on automatically. Sort option -r will be ignored
Reverse the order of the sort to get reverse collating sequence oldest first, or smallest file size first depending on the other options given
Set output width to the number of columns. 0 means no limit
